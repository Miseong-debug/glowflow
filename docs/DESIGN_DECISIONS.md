# Design Decisions

> 이 문서는 AESTHETIC CARE 앱의 설계 결정과 그 이유를 기록합니다.

---

## 1. 태블릿 퍼스트 설계 (Tablet First UI)

기존의 상단 네비게이션을 과감히 제거하고, 화면 활용도와 몰입도를 극대화하는 **"좌측 사이드 네비게이션(Side Nav)"**을 도입했습니다. 태블릿 기기에서 엄지손가락으로 탭을 옮기기에 최적화되었습니다.

**Why:**
- 에스테틱샵에서 관리사가 태블릿을 들고 고객 옆에서 기록하는 환경
- 상단 네비게이션은 세로 공간을 차지하여 인체도/차트 영역이 좁아짐
- 좌측 사이드 네비는 가로 방향 태블릿에서 자연스러운 엄지 접근성 제공

---

## 2. 벤토 박스 (Bento Box) 스타일 도입

각 영역, 고객 카드, 분석 결과 등을 둥글고 여유로운 벤토 박스 형태로 분리해, 직관적이고 최신 앱 같은 세련된 쉐도우(Shadow) 효과를 적용했습니다. 딱딱한 선 대신 매우 연한 색감의 보더와 부드러운 그림자를 사용했습니다.

**Why:**
- 고객이 화면을 함께 볼 때 "고급스러운 기술 앱"이라는 인상을 주기 위함
- 정보를 시각적 그룹으로 묶어 한눈에 파악 가능
- Apple 스타일의 모던 UI 트렌드 반영

---

## 3. 완전한 무테(Borderless) & 플로팅(Floating) 미학

요소 사이에 과도한 선(border)을 긋지 않고 여백(gap)으로 구분하며, '새 고객 추가' 등의 액션 버튼을 둥근 플로팅 버튼(FAB)으로 분리하여 훨씬 입체적인 조작감을 주었습니다.

**Why:**
- 선이 많으면 딱딱하고 구식 앱 느낌
- 여백과 그림자만으로 구분하면 시각적 노이즈가 줄어들어 핵심 정보에 집중
- FAB 버튼은 항상 접근 가능한 위치에 떠 있어 터치 효율성 극대화

---

## 4. 마이크로 애니메이션 강화

탭 이동 시의 아주 짧은 애니메이션, 버튼을 탭하거나 고객 카드를 클릭할 때 살짝 눌리는 듯한 반응형(active state) 스케일링 액션 등을 추가해 조작할 때의 손맛과 고급스러움을 한층 끌어올렸습니다.

**Why:**
- 터치 피드백이 없으면 "눌렀는지 안 눌렸는지" 불확실 → 사용자 불안
- 과하지 않은 미세한 애니메이션은 고급스러움의 핵심 요소
- 네이티브 앱 수준의 반응성을 웹에서 구현하여 "앱처럼" 느끼게 함

---

## 5. 관리 구분: Facial / Body 분리

고객 선택 후 Facial과 Body를 분리된 진입점으로 제공합니다.

**Why:**
- 페이셜은 얼굴+데콜테, 바디는 전신 → 관리 영역과 관심사가 완전히 다름
- 피부 측정기 데이터(수분/유분/탄력)는 주로 페이셜에 해당
- 바디는 미용(웨딩)과 체형교정으로 다시 나뉘어 별도 UI 필요
- 한 화면에 다 넣으면 복잡 → 모드 분리로 집중도 향상

---

## 6. 기록 UI 원칙: 간편 우선, 상세는 옵션

**Why:**
- 관리사마다 기록 스타일이 다름 (꼼꼼한 사람 vs 최소 기록 선호)
- 필수 입력을 많이 만들면 → 기록 자체를 안 하게 됨
- 탭/선택으로 빠르게 기록 + 필요 시 노트 추가 = 모든 관리사 커버
- 태블릿에서 키보드 타이핑은 불편 → 가능한 탭 위주 인터랙션

---

## 7. 색상 시스템: 인체도 톤 기반

로즈골드(#dca7a5) ~ 코퍼(#c4897e) 그라디언트를 앱 전체 액센트로 사용합니다.

**Why:**
- 인체도 라인 아트의 색상과 자연스럽게 어우러짐
- 에스테틱/뷰티 업계의 고급스러운 톤 (핑크골드, 로즈골드)
- 심각도 색상(초록→노랑→빨강)은 직관적인 신호등 체계로 별도 운영
- 글래스모피즘 + 로즈골드 = 미래적이면서 따뜻한 느낌
